## Base URL

```
https://silk.silkyway.ai/api
```

All endpoints are prefixed with `/api`. For local development, use `http://localhost:3000/api`.

## Authentication

The API is currently open for devnet usage. Authentication will be required for mainnet.

## Transaction Pattern

The API follows a consistent pattern for all Solana transactions:

<Steps>
  <Step title="Request an unsigned transaction">
    Call a `POST` endpoint (e.g., `/api/tx/create-transfer`). The backend builds an unsigned Solana transaction and returns it as base64.
  </Step>
  <Step title="Sign locally">
    Deserialize the transaction, sign it with your private key. **The backend never sees your private key.**
  </Step>
  <Step title="Submit the signed transaction">
    Call `POST /api/tx/submit` with the signed transaction. The backend submits it to Solana and waits for confirmation.
  </Step>
</Steps>

```typescript
// 1. Request unsigned transaction
const res = await fetch('/api/tx/create-transfer', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ sender, recipient, amount })
});
const { transaction, transferPda } = await res.json();

// 2. Sign locally
const tx = Transaction.from(Buffer.from(transaction, 'base64'));
tx.sign(keypair);
const signedTx = tx.serialize().toString('base64');

// 3. Submit
const submitRes = await fetch('/api/tx/submit', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ signedTx })
});
const { txid } = await submitRes.json();
```

## Response Format

All responses follow a consistent format:

**Success:**
```json
{
  "ok": true,
  "data": { ... }
}
```

**Error:**
```json
{
  "ok": false,
  "error": "Error message",
  "code": "ERROR_CODE"
}
```

## Rate Limits

| Endpoint Type | Limit |
| ------------- | ----- |
| Read (GET) | 100 requests/min |
| Write (POST) | 20 requests/min |
| Faucet | 1 request/10 min per wallet |
