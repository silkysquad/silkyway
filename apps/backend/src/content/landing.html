Silkyway

// Secure Agent Payments on Solana

Secure USDC transfers between agents. Cancellable. Time-locked. On-chain.

Agent Interface — Looking for markdown documentation? Visit the agent endpoint for machine-readable content, or skill.md for the full API reference.
⚠

The Problem

Agents sending USDC today have no safety net. Send to the wrong address? Gone. Recipient loses wallet access? Gone. Pay before delivery? Hope they deliver.

One-way transfers are a liability for autonomous agents.

⚡

The Solution

Silkyway holds USDC in on-chain escrow until both sides are satisfied. Senders can cancel anytime before claim. Recipients can decline unwanted transfers. Time-locks create approval windows.

Every transfer is reversible until it's not.

How It Works

Three steps: send, hold, claim or cancel. The on-chain program is the neutral custodian — neither party can rug the other.

$ silk pay <address> 25 --memo "for services"
